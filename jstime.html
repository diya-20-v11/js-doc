Introduction

JavaScript is the programming language of the web. It is the programming language that make the website interactive. It can update and change both HTML and CSS. It can calculate, manipulate and validate the data.

JavaScript runs in the browser through an engine that reads and executes the code.

When you open a webpage, the browser reads the HTML, applies the CSS, and then runs the JavaScript line by line using a built-in engine (like V8 in Chrome). This is how your code becomes actions on screen.

Analogy :

HTML builds the structure of a website, CSS makes it look good, and JavaScript adds life to it.

Code example is as follows :

 
     <script> 
     alert("Hello ! Welcome to my page"); 
     </script>

JavaScript is used in both frontend and backend development. When you use JavaScript for back-end, you will have to use Node.js. The script file or <script> tag which is in the HTML document will be shown in the browser.

Linking JavaScript to HTML

There are three ways to add JavaScript to an HTML document. They are as follows:

  1. Inline JavaScript

    You can add JavaScript directly inside an HTML element using event attributes like onclick, onmouseover, etc.

    
     <button onclick="alert('Hello!')">Click me</button>
  2. Internal JavaScript

    JavaScriptis written inside <script> tags, usually inside the <head> or just before the closing tag. 

     
 <!DOCTYPE html>
 <html>
  <head>
   <title>My Page</title>
  </head>
  <body>
   <h1>Hello World</h1>
    <script>
      // Your JavaScript code here
      console.log("Script runs just before the body closes");
    </script>
   </body>
 </html>
  3. External JavaScript

    You link a separate .js file in your HTML with a <script> tag's src attribute.

     
 <!DOCTYPE html>
 <html>
   <head>
     <title>My Page</title>
   </head>
   <body>
     <h1>Hello World</h1>
     <script src="script.js"> </script> 
   </body>
 </html>

    Here, script.js is your JavaScript file containing all your code.

When to use which method?

Situation Method
Quick, tiny JS for a specific element Inline JavaScript
Small script specific to one page Internal JavaScript
Medium to large apps, reusable or shared code External JavaScript(js file)

Variables

Variables are used to store data in a program. They act like containers that hold values. You can give them names to identify what they store. Variables can be changed or updated later. They help you reuse and manage data easily.

Rules for Variable names

  1. Variable names can include letters, digits, $, and _.
  2. They must not start with a digit.
  3. They cannot be JavaScript reserved keywords (like let, var, function ).
  4. Names are case-sensitive (name and Name are different).
  5. Use meaningful names that describe the data stored.
  6. Spaces and special characters (like @, #, !) are not allowed.

Declaring a variable

There are three ways to declare a variable. They are as follows :

  1. var - Old way, function-scoped 
    
 var name = "Alice";
  2. let - Modern way, block-scoped 
    
 let age = 25;
  3. const - Constant value, block-scoped 
    
 const country = "India";

In JavaScript, variable names, also identifiers, are used to store data values.

JavaScript Data types

JavaScript data types define the kind of value a variable can hold. They help the JavaScript engine understand how to store and use the data. Each value in JavaScript belongs to a particular data type. Data types make it easier to perform specific operations on different kinds of values.

Understanding data types is important for writing correct and bug-free JavaScript code.

Primitive Data types

A primitive data type in JavaScript holds a single, simple value and is stored directly in memory.

  1. Number : 
    
  let a = 5;
  2. String : 
    
  let name = "Harry";
  3. Boolean : 
    
  let isTrue = true;
  4. Undefined : 
    
  let x; // not assigned yet
  5. Null : 
    
  let y = null;
  6. BigInt : For very large integers
  7. Symbol : Unique and immutable values (used for object keys)

Non Primitive Data types

A non-primitive data type (also called a reference type) stores a reference to the memory location where the data is kept, not the actual value itself.

  1. Objects
  2. Arrays
  3. Functions
